PowerShell — progressif, guidé puis autonome

Automatisation, fichiers, utilisateurs locaux et diagnostics réseau avec PowerShell

Dans ce labo, toute l’activité se fait sur une seule machine virtuelle Windows. Les tâches sont conçues pour être réalisables, sûres et progressives. Les parties 1 à 4 représentent 60 % de la note, la partie 5 représente 20 %, la partie 6 représente 10 % et le QCM final représente 10 %.

Objectifs du labo

  • Préparer un environnement de travail sûr dans un dossier unique.
  • Créer simplement des groupes et des utilisateurs locaux à partir d’un fichier CSV.
  • Observer une vraie tâche d’automatisation pendant que le script tourne et capturer le résultat.
  • Produire un diagnostic local exploitable sur la même machine virtuelle.
  • Réaliser ensuite des tâches autonomes déjà basées sur des manipulations pratiquées en classe.
  • Documenter clairement le travail réalisé et répondre à un QCM final.

Partie 1 — Démarrage, sécurité et préparation du labo (60 % avec les parties 2 à 4)

Contexte et règles

Ce labo est conçu pour une seule VM Windows. Toutes les manipulations restent dans un dossier de travail pédagogique. Les actions demandées sont sûres : création de dossiers, création d’utilisateurs locaux simples, tri de fichiers dans des dossiers de test, diagnostic réseau local, batch de lancement, recherche de doublons et archivage de livrables.

Barème : les parties 1 à 4 représentent ensemble 60 % de la note.
Important : toutes les manipulations demandées sont locales, sûres et réalisables dans le dossier de travail du labo.

Préparer la structure du labo

Cette étape est entièrement guidée. Exécutez le bloc suivant dans PowerShell.

Partie 3 — Bloc guidé : création locale des utilisateurs et groupes (60 % avec les parties 1 à 4)

Écrire puis exécuter le script de création

Enregistrez ce script dans scripts\creer_utilisateurs_groupes.ps1, puis exécutez-le dans PowerShell.

creer_utilisateurs_groupes.ps1
PS C:\Users\Etudiant>
1$LabRoot = "$env:USERPROFILE\Documents\Labo_VM_Nom-Etudiant"
2$CsvPath = Join-Path $LabRoot "data\utilisateurs.csv"
3$LogPath = Join-Path $LabRoot "logs\creation_utilisateurs.log"
4
5if (-not (Test-Path $CsvPath)) {
6 Write-Host "Fichier CSV introuvable : $CsvPath" -ForegroundColor Red
7 exit
8}
9
10if (-not (Test-Path (Split-Path $LogPath))) {
11 New-Item -ItemType Directory -Path (Split-Path $LogPath) -Force | Out-Null
12}
13
14$utilisateurs = Import-Csv -Path $CsvPath
15
16$groupesExistants = @()
17$groupListRaw = net localgroup
18
19foreach ($line in $groupListRaw) {
20 if ($line -match "^\*") {
21 $groupesExistants += $line.TrimStart("*").Trim()
22 }
23}
24
25foreach ($u in $utilisateurs) {
26 $nom = $u.NomUser.Trim()
27 $motDePasse = $u.Password.Trim()
28 $groupe = $u.Groupe.Trim()
29
30 if ([string]::IsNullOrWhiteSpace($nom) -or
31 [string]::IsNullOrWhiteSpace($motDePasse) -or
32 [string]::IsNullOrWhiteSpace($groupe)) {
33 Add-Content -Path $LogPath -Value "$(Get-Date -Format s) | Ligne ignorée | données manquantes"
34 continue
35 }
36
37 if (-not ($groupesExistants -contains $groupe)) {
38 cmd /c "net localgroup `"$groupe`" /add" | Out-Null
39 $groupesExistants += $groupe
40 }
41
42 cmd /c "net user `"$nom`"" | Out-Null
43 $utilisateurExiste = ($LASTEXITCODE -eq 0)
44
45 if (-not $utilisateurExiste) {
46 cmd /c "net user `"$nom`" `"$motDePasse`" /add" | Out-Null
47 }
48
49 cmd /c "net localgroup `"$groupe`" `"$nom`" /add" | Out-Null
50
51 Add-Content -Path $LogPath -Value "$(Get-Date -Format s) | $nom | $groupe | créé/ou vérifié"
52 Write-Host "Traitement de $nom terminé" -ForegroundColor Cyan
53
54 Start-Sleep -Seconds 1
55}
56
57Get-Content $LogPath
Validation immédiate : vérifiez au moins un groupe local, un utilisateur local et le contenu du fichier journal creation_utilisateurs.log.

Partie 4 — Bloc guidé : automatisation observable et diagnostic local (60 % avec les parties 1 à 4)

Automatisation observable pendant l’exécution

Cette tâche doit permettre d’observer l’automatisation pendant l’exécution. Le script ci-dessous surveille le dossier a_traiter, puis range automatiquement les fichiers par type. Faites un premier essai avec depot_source, puis un deuxième essai avec depot_source2, afin d’observer le résultat sur place deux fois.

tri_observable.ps1
PS C:\Users\Etudiant>
1$LabRoot = "$env:USERPROFILE\Documents\Labo_VM_Nom-Etudiant"
2$WatchFolder = Join-Path $LabRoot "a_traiter"
3$LogPath = Join-Path $LabRoot "logs\tri_observable.log"
4$DepotSource = Join-Path $LabRoot "depot_source"
5$DepotSource2 = Join-Path $LabRoot "depot_source2"
6$ImagesFolder = Join-Path $WatchFolder "Images"
7$DocumentsFolder = Join-Path $WatchFolder "Documents"
8$TextesFolder = Join-Path $WatchFolder "Textes"
9New-Item -Path $ImagesFolder -ItemType Directory -Force | Out-Null
10New-Item -Path $DocumentsFolder -ItemType Directory -Force | Out-Null
11New-Item -Path $TextesFolder -ItemType Directory -Force | Out-Null
12"Lancez ce script dans une console PowerShell indépendante et laissez la fenêtre ouverte." | Set-Content -Path $LogPath
13Write-Host "Console de surveillance active. Faites un premier essai avec depot_source puis un deuxième avec depot_source2." -ForegroundColor Yellow
14while ($true) {
15 $files = Get-ChildItem -Path $WatchFolder -File -ErrorAction SilentlyContinue
16 foreach ($file in $files) {
17 switch ($file.Extension.ToLower()) {
18 ".jpg" { $dest = $ImagesFolder }
19 ".png" { $dest = $ImagesFolder }
20 ".pdf" { $dest = $DocumentsFolder }
21 ".docx" { $dest = $DocumentsFolder }
22 ".txt" { $dest = $TextesFolder }
23 ".log" { $dest = $TextesFolder }
24 default { $dest = $WatchFolder }
25 }
26 if ($dest -ne $WatchFolder) {
27 Move-Item -Path $file.FullName -Destination (Join-Path $dest $file.Name) -Force
28 Write-Host "$($file.Name) déplacé vers $(Split-Path $dest -Leaf)" -ForegroundColor Green
29 Add-Content -Path $LogPath -Value "$(Get-Date -Format T) | $($file.Name) -> $(Split-Path $dest -Leaf)"
30 Start-Sleep -Seconds 1
31 }
32 }
33 Start-Sleep -Seconds 2
34}
Procédure de test : lancez ce script dans une console PowerShell indépendante et laissez la fenêtre ouverte. Divisez ensuite l’écran en 3 parties : la console PowerShell, un dossier source et le dossier a_traiter. Faites un premier essai en copiant les fichiers de depot_source vers a_traiter, puis faites un deuxième essai avec depot_source2. Observez le tri automatique dans l’Explorateur Windows pendant les deux essais. Arrêtez le script seulement après les captures, avec Ctrl + C.

Produire un diagnostic local sûr sur la même VM

Le diagnostic suivant reste entièrement local et ne dépend d’aucune deuxième machine virtuelle.

diagnostic_vm.ps1
PS C:\Users\Etudiant>
1$LabRoot = "$env:USERPROFILE\Documents\Labo_VM_Nom-Etudiant"
2$ReportPath = Join-Path $LabRoot "rapport\diagnostic_vm.txt"
3"=== diagnostic de la VM ===" | Set-Content -Path $ReportPath
4"--- configuration IP ---" | Add-Content -Path $ReportPath
5Get-NetIPConfiguration | Format-List InterfaceAlias,IPv4Address,IPv4DefaultGateway,DNSServer | Out-String | Add-Content -Path $ReportPath
6"--- test loopback ---" | Add-Content -Path $ReportPath
7Test-Connection 127.0.0.1 -Count 2 | Out-String | Add-Content -Path $ReportPath
8"--- résolution DNS ---" | Add-Content -Path $ReportPath
9Resolve-DnsName localhost | Out-String | Add-Content -Path $ReportPath
10"--- ports en écoute ---" | Add-Content -Path $ReportPath
11Get-NetTCPConnection -State Listen | Select-Object -First 10 LocalAddress,LocalPort,OwningProcess | Format-Table -AutoSize | Out-String | Add-Content -Path $ReportPath
12Get-Content $ReportPath

Partie 5 — Tâches de recherche (20 %)

Barème : cette partie représente 20 % de la note.

Consignes générales du bloc autonome

Dans cette partie, vous devez retrouver et appliquer la bonne méthode à partir des notions déjà pratiquées en classe. Les tâches demandées reprennent des manipulations déjà réalisables sur la machine virtuelle du labo.

Règle : les solutions attendues doivent rester réalisables, sûres et cohérentes avec les manipulations déjà pratiquées en classe.

Mini-défis à réaliser

Tâche autonome 1 — rechercher les doublons avec Get-FileHash

  • Créer scripts\recherche_doublons.ps1.
  • Le script doit analyser le dossier photos du labo.
  • Il doit produire un fichier rapport\doublons.txt où l’on voit clairement quels fichiers ont le même hash.
  • Utiliser -WhatIf seulement si vous voulez simuler une suppression, mais aucune suppression n’est obligatoire ici.
Indices autorisés :
  • Get-ChildItem "photos" -File -Recurse
  • Get-FileHash
  • Group-Object Hash
  • Where-Object { $_.Count -gt 1 }
  • Out-String + Set-Content

Tâche autonome 2 — ranger automatiquement les fichiers par type

  • Créer scripts\rangement_par_type.ps1.
  • Le script doit traiter le dossier downloads_test.
  • Il doit créer des sous-dossiers cohérents, puis y déplacer les fichiers selon leur extension.
  • À la fin, le dossier downloads_test doit clairement montrer le rangement effectué.
Indices autorisés :
  • $TargetFolder
  • Get-ChildItem -File
  • switch ($file.Extension.ToLower())
  • New-Item -ItemType Directory
  • Move-Item

Tâche autonome 3 — créer deux lanceurs batch

  • Créer lancer_creation.bat pour lancer le script de création des utilisateurs.
  • Créer lancer_diagnostic.bat pour lancer le script de diagnostic local.
  • Les deux fichiers doivent fonctionner même si le dossier du labo est déplacé.
Indices autorisés :
  • powershell -ExecutionPolicy Bypass -File
  • %~dp0
  • pause

Tâche autonome 4 — préparer le dossier d’archives de remise

  • Créer un sous-dossier de remise dans archives avec un nom explicite et horodaté.
  • Copier dans ce dossier au minimum : README_TECHNIQUE.md, diagnostic_vm.txt, doublons.txt et les fichiers de log utiles.
  • Le résultat doit être facile à corriger et à relire.
Indices autorisés :
  • New-Item
  • Join-Path
  • Get-Date -Format yyyyMMdd-HHmm
  • Copy-Item
  • Get-ChildItem
